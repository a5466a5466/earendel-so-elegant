## 實作範圍

- 直接讀取既有 `events` Content Collection，沒有建立第二份搜尋資料或複製活動欄位。
- 支援標題、摘要、主題、標籤與媒體名稱的關鍵字搜尋，文字先以 NFKC 正規化並轉為小寫。
- 支援年份、單一標籤與媒體類型篩選；媒體類型由 Gallery、Video、Audio 與 socialPosts 陣列推導，不新增 schema 欄位。
- 所有非空條件寫入 `q`、`year`、`tag`、`media` query parameters；直接網址、重新整理與 `popstate` 都會恢復表單與結果。
- 關鍵字輸入使用短 debounce 並 replace 目前 History；選單、送出與清除使用 push，讓返回鍵可比較前後條件而不替每個字建立紀錄。
- 顯示結果數、目前條件、清除全部與無結果狀態；無效或缺少 tags／media data attribute 會安全視為空陣列。
- 篩選器使用原生 `<details>`、`<form>`、search input 與 select；手機可收合，鍵盤與觸控不需自訂控制器。
- 初始 HTML 永遠輸出完整三筆活動；JavaScript 關閉時不執行篩選，但內容仍可閱讀並有 noscript 說明。
- 加入 Lab 導覽、麵包屑、實驗索引入口與響應式樣式；正式建置移除搜尋路由及 bundle。

## 套件決策

目前只有三筆活動且都在 2026 年，原生陣列過濾已足夠，Pagefind 沒有投入價值。本 Step 不安裝套件；等活動超過約 20 筆、需要跨頁內文索引、詞彙權重、分頁結果或多語全文搜尋時，再另開 dependency 核准。

## 人工驗收重點

1. 開啟 `/lab/search/`，確認初始顯示三筆活動與「共 3 筆結果」。
2. 搜尋「生日」，確認只剩星光生日企劃，網址含 `q`；重新整理後條件仍保留。
3. 清除後選擇標籤「線上應援」，確認有 2 筆；再選「圖片 Gallery」，確認只剩盛夏來信。
4. 使用瀏覽器返回鍵，確認回到只有「線上應援」且結果恢復為 2 筆。
5. 輸入不存在的詞，確認顯示 0 筆與無結果說明；按「清除全部」回到完整列表。
6. 測試年份 2026、標籤、媒體和關鍵字的任意組合，確認結果數與卡片一致。
7. 複製含條件的網址到新分頁，確認表單、結果與目前條件摘要一致。
8. 以手機寬度收合／展開「搜尋與篩選條件」，確認長文字、控制器與卡片沒有溢出。
9. 若方便，可暫時停用 JavaScript 或檢視初始 HTML，確認完整三筆活動仍存在。
10. 確認目前資料量不需要 Pagefind；若接受，核准維持原生方案並延後套件。
