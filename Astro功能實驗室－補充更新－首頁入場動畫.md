# Astro 功能實驗室・補充更新・首頁入場動畫

- 建立日期：2026-07-23
- 目前狀態：規劃完成，尚未開始實作
- 類型：Phase 1～4 結案後的補充更新
- 目標頁面：正式首頁 `/`
- 參考網站：`https://cyg.sid.tw/`
- Dependency：預計使用原生 CSS 與少量 TypeScript，不新增套件

## 1. 文件目的

本文件管理正式首頁的入場動畫補充需求。此工作不回寫或重開既有 Phase 1～4，也不納入已結案 Step；後續設計、實作、QA、人工驗收與部署紀錄都集中追加於本文件。

## 2. 參考效果觀察

2026-07-23 已實際重新載入參考網站並分段觀察：

1. 畫面先以全黑背景開始。
2. 頂部導覽逐漸淡入。
3. 中央顯示大型白色主標文字。
4. 右下角小角色與主標在相近階段進場。
5. 約一至兩秒後主標退場，首頁主要內容接續揭露。

本站只參考「全螢幕、分階段揭露、主標退場後進入首頁」的敘事節奏，不複製對方文字、圖片、角色、品牌識別或程式碼。

## 3. 本站候選版本

預定流程：

```text
深色全螢幕
→ 星光或厄倫蒂兒識別元素出現
→ 厄倫蒂兒主標題進場
→ 主標題退場
→ 正式首頁 Hero 與導覽揭露
```

視覺方向沿用本站既有深藍、金色、星光與典雅字體，不建立與參考網站相同的黑白視覺複本。正式文案、識別素材與時間曲線需在實作前由使用者確認。

## 4. 第一版範圍

- 只處理正式首頁首次載入的視覺揭露。
- 使用原生 CSS animation／transition 與少量 TypeScript 控制完成狀態。
- 首頁主要 HTML 必須預先存在，不等待動畫才建立內容。
- 動畫層不得阻擋頁面載入失敗時的閱讀與導覽。
- 動畫完成後必須完整移除覆蓋層與互動攔截。
- 暫不加入影片、音訊、Canvas、WebGL、Live2D 或新 dependency。

## 5. 待決策項目

- 入場動畫每次重新整理都播放，或只在同一瀏覽工作階段首次進站播放。
- 主標文案與是否加入副標。
- 使用純文字、現有星光圖形，或另製正式識別素材。
- 總長度、各階段節奏與略過方式。
- 手機版是否縮短時間或減少元素，但不得自行切換成低效能模式。

## 6. 必要行為與防護

- 使用者選擇「減少動態」時，改為短淡入或直接顯示首頁。
- 提供可被鍵盤與指標操作的「略過動畫」方式，或允許任意互動立即完成；最終形式於實作時確認。
- 動畫期間不得鎖死返回、重新整理、網址列或瀏覽器操作。
- JavaScript 關閉、CSS 載入失敗或 runtime 例外時，首頁內容仍直接可見。
- 避免造成 layout shift；覆蓋層使用固定定位，首頁本體維持正常版面。
- 不自動播放聲音；若未來加入聲音，必須另開需求並遵守使用者手勢限制。

## 7. 實作順序

1. 確認主標文案、識別素材、是否每次播放與總長度。
2. 建立純 CSS／HTML 靜態原型，確認首尾畫面。
3. 加入階段控制、略過與完成後清理。
4. 驗證桌機與手機版，不影響首頁內容、導覽與既有 Lab 入口。
5. 執行 Lab／production build、瀏覽器 Console 與動畫時間 QA。
6. 使用者人工確認節奏與視覺後，更新本文件、`handoff.md`、commit 並 push。

## 8. 驗收條件

- 入場流程清楚但不拖延進站，總時間符合使用者核准值。
- 主標、首頁 Hero 與導覽的出場／進場順序穩定。
- 動畫完成或略過後，不殘留透明遮罩、焦點陷阱、scroll lock 或 listener。
- 360px、768px、1440px 無溢位或文字裁切。
- 重新整理、返回首頁、深層頁返回與 JavaScript 關閉皆可正常使用。
- Console error／warning 為 0，建置與 GitHub Pages Repository base 路徑通過。
- 使用者完成人工驗收後才標記完成。

## 9. 目前結論

本輪只完成需求建檔與交接更新，尚未修改首頁程式碼。下一步先由使用者確認第 5 節的播放頻率、文案、素材與時間，再開始實作。
