## 實作範圍

- 沿用 Step 14 的 `audio-manager.ts`，加入單一短音效 channel；沒有為每個按鈕建立獨立 `Audio` 實例。
- 以 Node 內建 API 產生成功、取消與 hover 三個單聲道 WAV，長度分別為 0.24、0.20、0.09 秒。
- 成功與取消按鈕永遠先更新可見文字與操作次數，再嘗試播放聲音；靜音或播放失敗不影響結果表達。
- 點擊操作會解鎖本頁短音效；hover 在尚未互動前不播放，觸控 pointer 不觸發 hover。
- 快速觸發使用單一 channel 重啟與 70ms action cooldown，不讓五個音源疊加；hover 使用 320ms cooldown。
- 節能模式保留明確成功／取消聲音，但略過 hover；短音效音量由 0.32 降為 0.22。
- 「關閉並記住音效設定」可由同一按鈕重新開啟；它與「重設所有偏好」直接使用既有 F26 preferences API 與 `earendel-lab-preferences-v1`，沒有第二個儲存鍵。
- 偏好切回 off、頁籤隱藏、切頁、重新初始化與 dispose 都會立即停止短音效。
- 不安裝 Howler；正式站也不建議替所有導覽與 hover 全面加入聲音。

## 暫定整合決策

目前採原生 `Audio`、一個共用 effect channel 與既有 F26 偏好狀態，不安裝 Howler。這個方案已涵蓋短音效切換、快速觸發節流、節能降級、永久靜音與生命週期清理。若未來出現大量同時音源、精確混音、sprite、跨場景淡入淡出或遊戲級延遲需求，再另行評估 Web Audio API 或 Howler。

## 人工驗收重點

1. 開啟 `/lab/sound-effects/`，保持音效為「關閉」，按成功／取消；不應出聲，但結果文字與次數必須更新。
2. 將頁首音效改為「開啟」；偏好切換本身不應出聲。按「確認保存」與「取消修改」，確認兩種短音不同且音量舒適。
3. 尚未點擊本頁操作前滑入按鈕不應出聲；點擊一次後再滑入，hover 聲應短且有 320ms 節流。
4. 按「執行快速觸發」，確認不會出現五層疊音或突然變大聲。
5. 將效能模式改為「節能」，確認點擊聲保留、hover 聲略過且狀態有說明。
6. 按「關閉並記住音效設定」，重新整理後應保持靜音；同一按鈕應顯示「重新開啟音效」，按下後可立即恢復。
7. 按「重設所有偏好」，確認音效回到安全預設「關閉」、計數歸零。
8. 播放短音效時切換頁籤或離開頁面，不應殘留或返回後自動播放。
9. 以手機寬度檢查兩張操作卡、結果區與偏好按鈕沒有溢出。
10. 判斷原生短音效是否足夠；若足夠，正式核准不安裝 Howler。
